Welcome to the API Documentation

To build the server­side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

API Endpoints

Business Logic/What does it do? Endpoint URL Query Parameters Method Type Request Data Format Example Request Response Data Format Example Response
Get List of All Movies /movies N/A GET Nothing in body of request N/A A JSON object holding data about all movies See "Get data about single movie by title" for each item format.
Get data about single movie by title /movies /:Title GET Nothing in body of request N/A A JSON object holding data about a single movie, also containing an ID. { _id: '5ff8a361ccf09d1cd55495f9', title: 'Arrival', description: 'A linguist works with the military to communicate with alien lifeforms after twelve mysterious spacecrafts appear around the world.', year: 2016, genre: 'Drama' director: { 'Dennis Villeneuve' : 'Denis Villeneuve OC CQ is a French Canadian film director, producer, and screenwriter, etc.' '1967-10-03' } image URL: 'https://www.imdb.com/title/tt2543164/mediaviewer/rm3938516992/' True }
Get data about genre by name/title (Ex: "Thriller") /movies/genres /:Name GET Nothing in body of request N/A A JSON object holding description of genre. { Thrillers: This genre of movies are characterized and defined by the moods they elicit, giving viewers heightened feelings of suspense, excitement, surprise, anticipation and anxiety. }
Get data about a director /movies/directors /:Name GET Nothing in body of request N/A A JSON object holding data about a single director. { Dennis Villeneuve: Dennis Villeneuve OC CQ is a French Canadian film director, producer, and screenwriter. He is a four-time recipient of the Canadian Screen Award for Best Direction...etc. }
Allow new users to register /users N/A POST A JSON object holding data about the user to add. { "username": "KSFlynn007", "password": "Password123", "birthday": "April 7, 1994", "email": "testing@gmail.com" } A JSON object holding data about the user that was added, including a generated ID#. { id: 27, username: 'KSFlynn007', password: Password123, birthday: April 7, 1994, email: testing@gmail.com }
Allow users to update their user info(username) /users /:Username PUT Nothing in body of request N/A A JSON obejct holding updated data about user. { id: 27, username: 'SciFiFan007', password: Password123, birthday: April 7, 1994, email: testing@gmail.com }
Allow users to add a movie to their favorite list /users/:Username /:FavoriteMovies/:MovieID POST Nothing in body of request The MovieID in Query Parameters (:MovieID) will add the correct data into the FavoriteMovies array bu accessing the Movies collection. Ex; ObjectID("5ff8b3168ac2b2804e15411b"). A JSON object holding returned updated user data. { id: 27, username: 'SciFiFan007', password: Password123, birthday: April 7, 1994, email: testing@gmail.com, Favorite Movies: [ObjectId("5ff8b3168ac2b2804e15411b")] }
Allow users to remove a movie from their list of favorites /users/:Username /:FavoriteMovies/:MovieID DELETE Nothing in body of request N/A A JSON object holding returned updated user data. { id: 27, username: 'SciFiFan007', password: Password123, birthday: April 7, 1994, email: testing@gmail.com, Favorite Movies: [ ] }
Allow users to deregister /users /:Username DELETE Nothing in body of request N/A A text message confirming deregistration of user. '[Username] was deleted.'
